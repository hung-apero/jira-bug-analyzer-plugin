/jira-bug-analyzer — sơ đồ luồng

Agent harness-loop: kéo bug Jira → phân tích ảnh/video bằng thị giác → chấm điểm tin cậy → sửa trong worktree riêng → kiểm thử trên máy → mở PR. Học vào KB Android dùng chung + chống fix trùng cùng root_cause_slug. Mọi I/O bộ nhớ remote chạy NỀN, không chặn luồng chính.

Manager — --manager <KEY> (chạy sau batch, KHÔNG phải pipeline)
Job Agom & xử lý PR review-comment (--auto: tự áp dụng fix rõ ràng)
Job Bdọn worktree khi PR merge / đóng
Job CKB-backfill ticket đã Done — re-judge tri thức độc lập app (tự động)
Tham số
--auto vòng tự lái hoàn toàn (pick→fix-vs-defer→verify→PR nền→kéo lại). Mặc định TẮT = mọi cổng tương tác.
--manager chế độ 3: reconcile board (review-comment + worktree + KB-backfill), 1 lượt. Cần <KEY>.
--team multi + Agent Team (Lead+Devs+Tester+Observer); --devs N đặt số Dev.
@N chọn phase tường minh (vd @3) — thắng suy luận; không có thì suy từ tiêu đề spec.
--model opus|sonnet|haiku ghi đè ma trận model theo từng bước.
--discord bật đăng review PR lên Discord (mọi mode; với --auto là cách duy nhất — mặc định TẮT). Cần PR_DISCORD_CHANNEL_URL.
--resume đồng bộ lại theo dõi PR / trạng thái chờ (modifier).
--recheck-env buộc chạy lại env preflight, bỏ cache.
--prune <PROJ> xóa bộ nhớ phiên (session/ ledger) khi đóng phase; KB/setup giữ.
--pr-window ĐÃ BỎ — review stream theo từng ticket.
--every ĐÃ BỎ — multi tự kéo lại mỗi lượt.
Quy tắc luôn đúng
[CLAIM] claim → In-Progress + gán trước khi phân tích [VERIFY] review + user-test pass mới PR [1COMMIT] 1 commit / ticket, không squash [VN] hiển thị tiếng Việt · PR & bộ nhớ tiếng Anh [OPTIONS] luôn cho bấm chọn [REST] Jira ghi qua REST + đọc lại xác nhận [BGMEM] bộ nhớ remote chạy nền [SETUP] ghi setup.json qua setup-json.sh merge-set [WIRE] auto-wire bộ nhớ trước pull/claim [BASE] luôn lấy code mới nhất origin/<BASE> [adb] test bằng adb, không mobile-mcp
Bộ nhớ dùng chung = repo GitHub riêng tư (mỗi entry 1 file markdown): KB Android + ledger trạng thái project/<PROJ>/<PHASE>/session/record_bug_status.md. Cách dùng chi tiết: usage-by-mode.md.
